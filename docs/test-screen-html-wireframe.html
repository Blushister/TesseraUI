Acceptance Criteria — Semantic HTML

FeatureGiven / WhenExpectedBug symptom
<p> wraps <p>Long text...</p> in a column narrower than the text Text splits into multiple lines; label height grows accordingly Text overflows on one line / label height stays at one line
white-space:normal on <label> <label style="white-space:normal">Long...</label> Label wraps exactly like <p> No wrap / label stays single-line
Plain <label> no wrap <label>Long text…</label> without white-space Text stays on one line, clipped at widget boundary Text wraps unexpectedly
Wrap pushes siblings down Wrapping label followed by another widget in a column Sibling is placed below the full wrapped height, no overlap Sibling overlaps the wrapped label
h1 size <h1>Heading 1</h1> with no CSS font-size override Text renders at 14px, bold Same size as body text / not bold
h2–h6 progressive sizes <h2> to <h6> stacked in a column Sizes: 12 / 10 / 9 / 8 / 7px, all bold, visibly decreasing All same size / not bold
Heading CSS override h2 { font-size: 7px; font-weight: normal; } h2 renders at 7px, not bold — CSS overrides UA default Still renders at 12px / still bold
<ul> bullet prefix <ul><li>Item</li></ul> Each <li> renders with "• " prefix No prefix / number prefix instead of bullet
<ol> number prefix <ol><li>A</li><li>B</li></ol> <li> items show "1. " and "2. " respectively Bullet instead of number / all items show "1. "
<li> wraps long text <li> with text longer than column width Text wraps; prefix stays on first line; height grows Text overflows / sibling overlaps
<strong> / <b> <strong>Bold</strong> Text rendered with font-weight 700 Same weight as regular text
<em> / <i> <em>Italic</em> Text rendered in cream-dim color (#C2AD8E) as italic hint Same color as regular text / crashes
<a> transparent button <a onclick="handler">Link</a> Button with transparent background, copper-hi color, clickable Solid background / wrong color / handler not fired
<section> as column container <section><p>...</p></section> Section acts as column panel; <p> inside wraps normally Section tag ignored / children not rendered
flattenText for mixed content <p>Hello <strong>World</strong></p> Paragraph renders "Hello World" (bold styling merged into text) Only "Hello" shown / "World" lost
