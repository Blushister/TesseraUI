Acceptance Criteria — border-radius

Feature Given / When Expected Bug symptom
Rounded corners appear border-radius: 4px on a div with background The four corners of the fill are clipped; the panel visually has rounded corners Panel still renders as a sharp rectangle / corners not clipped
Radius = 0 is identical to no radius border-radius: 0 (or no property set) Panel renders as a plain rectangle, no performance difference Small corner arcs appear / flickering at corners
Radius clamped to half dimension border-radius: 100px on a 40×20 panel Radius clamped to 10 px (half of min dimension); renders as a pill shape Radius overflows / rendering glitch / negative arc
Scanline per-row accuracy Panel at small size (e.g. 20×10) with radius 4 Arc is smooth (no single-pixel spikes or gaps) at MC pixel resolution Jagged corners / stray filled pixels outside the arc boundary
Background fills inside arc Panel with background: #1F1812; border-radius: 5px, no border Background color fills the rounded rect; areas outside arc are transparent Background fills entire bounding box (square)
Border drawn as outer rounded rect Panel with border: 1px solid #D89255; border-radius: 5px Rounded border traces the outer arc; background inset by 1 px fills inner arc Border is square / border outside rounded bg / seam visible
Thick border inner radius reduced border: 2px solid …; border-radius: 6px Inner background arc has radius 4 (6 − 2); no gap between border and bg Gap between border arc and background / inner radius same as outer
Background-only (no border) still rounds Panel with border-radius: 4px, no border CSS Only the background rounded rect is drawn; no border artifact Thin phantom border rendered / extra 1px outline
border-radius parsed from CSS .card { border-radius: 4px } in template CSS TesseraStyle.borderRadius == 4 after parsing; panel gets the radius borderRadius stays UNSET / parsed as 0 / ignored in renderer
border-radius parsed without px unit border-radius: 6 (bare integer) Parsed as 6 px, same result as 6px Parsed as 0 / exception thrown
border-radius inherits through TesseraStyle.merge() Base style has borderRadius = 3; override style leaves it UNSET Merged style keeps borderRadius = 3 from base Merged style resets borderRadius to UNSET / to 0
border-radius in CSS variable :root { --r: 5px } .card { border-radius: var(--r) } Variable resolved to 5; panel rendered with radius 5 Variable not resolved / borderRadius = 0
Square-border panels unaffected Panel with border: 1px solid … and NO border-radius Border renders as four flat g.fill() lines — unchanged from v1.0 Panel corners rounded when borderRadius was never set
Side borders (borderTop/Left/…) unaffected by radius Panel with border-top: 1px solid … and border-radius: 4px Side-specific border lines are suppressed (radius > 0 path skips square lines) Square border lines drawn on top of rounded arc / double border
Rounded cards in flex row Five cards with width:64px; border-radius:5px in a row All five cards visible side-by-side with rounded corners and correct accent border color Cards clipped / square corners / border color missing
Children inside rounded panel not clipped Label inside a rounded card (no overflow:hidden) Label text renders normally; rounding applies only to background/border, not to child scissor Text cut by rounded clip / text invisible
