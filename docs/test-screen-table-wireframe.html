Acceptance Criteria — Table / Text-decoration / Img

Feature Given / When Expected Bug symptom
<table> column layout <table> wrapping <tr> rows Rows stacked vertically; table width fills available space Rows side-by-side / table width = 0 / crash
<thead> / <tbody> / <tfoot> transparency Rows nested inside section wrappers: <thead><tr>… Wrappers pass through; rows render identically to direct <tr> children Rows lost / extra blank spacing from wrapper
<td> equal column widths Three <td> inside one <tr>, no explicit width CSS Each cell occupies ≈ ⅓ of the row width (flex:1) All cells width=0 / only first cell visible / cells overflow
<th> bold + centered by default <th>Name</th> with no CSS override Header cell renders bold, text centered Same weight/alignment as <td>
Cell padding CSS td { padding: 2px 3px } Content inside cell has visible inset from cell border Content flush against edge / padding ignored
Row auto-height Cell contains wrapping text or tall content Row height expands to tallest cell; no content clipped Row fixed at one-line height; content cut off
Table auto-height Table with no explicit CSS height Table panel height = sum of row heights + gaps + padding Table collapses to naturalHeight (12px) / rows overflow
CSS on <td> / <th> td { background: #120F0A; color: #F3E7D3; } Each cell gets the declared background and text color CSS ignored; default panel appearance
CSS on <tr> tr { background: #1F1812; } Entire row gets the row background Row background ignored
text-decoration: underline on <p> <p class="ul"> with .ul { text-decoration: underline } A 1px line drawn immediately below the glyph baseline, same color as text No line / line drawn at wrong y / line wrong color
text-decoration: line-through on <p> .lt { text-decoration: line-through } A 1px line drawn through the vertical midpoint of the glyphs No line / line at bottom instead of middle
text-decoration on <label> <label class="lbl"> with .lbl { text-decoration: underline } Underline rendered on the single-line label path Underline missing on label / only works on <p>
text-decoration on wrapped text Label or <p> with white-space:normal AND text-decoration:underline Each line of the wrapped text has its own underline (line width matches text width per line) Only first line has decoration / decoration spans full widget width
text-decoration: none cancels decoration Parent has text-decoration:underline; child sets text-decoration:none Child element has no decoration line Child still shows decoration from parent
text-decoration on <span> <span class="sp"> with .sp { text-decoration: underline } Span label renders with underline Span decoration ignored
<img> loads ResourceLocation <img src="minecraft:textures/item/diamond.png" width="16" height="16"/> TesseraIcon rendered at 16×16 with the MC diamond texture Missing texture / crash / renders as 0×0
<img> namespace:path split src contains a colon: "minecraft:textures/…" Split at first colon → namespace=minecraft, path=textures/… Invalid ResourceLocation / no colon split
<img> bare path (no namespace) src="textures/foo.png" (no colon) Falls back to namespace=minecraft Crash on missing namespace
<img> explicit width / height width="32" height="16" Icon occupies exactly 32×16 screen pixels Icon always square / size ignored
<img> default size fallback <img> with no width or height attribute Renders at 16×16 px default Renders at 0×0 / crash
<img> color tint via CSS img { color: #FF8040 } Texture drawn with the declared ARGB tint color Tint ignored; texture always white/uncolored
