Acceptance Criteria — TesseraUI v1.2

  • AC-01 · textarea renders — background, 1px border, correct size from rows attribute
  • AC-02 · placeholder — shown in muted colour when empty + unfocused; hidden on focus or text entry
  • AC-03 · focus ring — border switches to focusBorderColor on click; restored on blur
  • AC-04 · text entry — characters append at cursor; Enter inserts newline; content split by \n
  • AC-05 · cursor navigation — ←/→ move by character; ↑/↓ move between lines; Home/End per-line; Ctrl+Home/End to document start/end
  • AC-06 · selection — Shift+navigation selects; selection highlighted per-line; Ctrl+A selects all
  • AC-07 · clipboard — Ctrl+C copies; Ctrl+X cuts; Ctrl+V pastes (preserving newlines)
  • AC-08 · vertical scroll — scrollY clamped every frame so cursor is always visible; scrollbar rendered when content overflows
  • AC-09 · blinking cursor — 500 ms on/off when focused; resets on any keystroke
  • AC-10 · CSS height — explicit height: overrides rows default
  • AC-11 · :focus pseudo-class — textarea:focus { border-color: … } wired to focusBorderColor
  • AC-12 · position:absolute routing — child with position:absolute is added via panel.addAbsolute(), not flex flow
  • AC-13 · absolute positioning — top/right/bottom/left resolve relative to padded parent; widget placed each frame
  • AC-14 · absolute overflow — absolute child renders outside scissor clip (drawn after normal children)
  • AC-15 · absolute hit-testing — absolute children receive mouseClicked before flex children (they're on top)
  • AC-16 · textarea tag — "textarea" present in TesseraNode.KNOWN_TAGS
  • AC-17 · /tessera test-v12 — command registered; opens TesseraTestScreenV12
  • AC-18 · state persistence — textarea state tied to inputStates map by id; survives panel rebuilds
