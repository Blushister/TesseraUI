Acceptance Criteria — TesseraUI v1.3

  • AC-01 · calc() width — width: calc(100% - Xpx) resolves against parent availW at layout time
  • AC-02 · calc() height — height: calc(N% ± Xpx) resolves against parent availH
  • AC-03 · calc() stored — expression stored in widthCalc/heightCalc; plain width field stays UNSET
  • AC-04 · calc() merge — TesseraStyle.merge() propagates widthCalc/heightCalc correctly
  • AC-05 · flex-grow — free space distributed proportionally to flex-grow weights
  • AC-06 · flex-grow rounding — integer-pixel rounding remainder given to last growing item
  • AC-07 · flex-shrink — overflow reduced proportionally to flex-shrink × baseSize
  • AC-08 · flex-shrink:0 — item with flex-shrink:0 never changes size on overflow
  • AC-09 · flex-basis — explicit px value used as starting size instead of content size
  • AC-10 · flex-basis auto — flex-basis: auto / UNSET → content width/height used as base
  • AC-11 · flex shorthand N — flex: N → grow=N, shrink=1, basis=0 (N>0) or basis=auto (N=0)
  • AC-12 · flex shorthand N M — flex: N M → grow=N, shrink=M, basis=0
  • AC-13 · flex shorthand N M Xpx — flex: 1 1 80px sets all three fields
  • AC-14 · flex: none / auto — keyword shorthands expand correctly
  • AC-15 · order row — items sorted by order value before row layout
  • AC-16 · order column — items sorted by order value before column layout
  • AC-17 · order stable — equal order values preserve DOM insertion order
  • AC-18 · justify-content unchanged — justify-content still works when no flex-grow items present
  • AC-19 · backward compat — old add(widget, int flex) API maps to grow/shrink/basis correctly
  • AC-20 · /tessera test-v13 — command registered; opens TesseraTestScreenV13
