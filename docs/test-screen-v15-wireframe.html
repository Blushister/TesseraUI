Acceptance Criteria — TesseraUI v1.5 (z-index)

  • AC-01 · CSS parsing — z-index: <n> parsé dans TesseraCssParser → TesseraStyle.zIndex (int, UNSET par défaut)
  • AC-02 · CSS merge — TesseraStyle.merge() propage zIndex correctement (other wins si != UNSET)
  • AC-03 · Entry.zIndex — le record TesseraPanel.Entry contient un champ int zIndex
  • AC-04 · render order — render() trie les enfants flux normal par zIndex ASC (stable) avant de les peindre
  • AC-05 · z-index default 0 — sans z-index CSS, zIndex=0 → l'ordre DOM est préservé (tri stable)
  • AC-06 · hit-test order — mouseClicked() itère en zIndex DESC : le widget visuellement au-dessus capte les clics en premier
  • AC-07 · AbsEntry.zIndex — le record AbsEntry contient aussi un champ int zIndex ; les enfants absolus sont triés ASC au rendu
  • AC-08 · renderer fZIndex — TesseraTemplateRenderer extrait fZIndex depuis childStyle.zIndex et le passe à panel.add(..., fZIndex, ...)
  • AC-09 · stack demo — les trois panneaux rouges / verts / bleus se superposent ; le bleu (z:10) peint au-dessus du vert (z:5), lui-même au-dessus du rouge (z:1)
  • AC-10 · badge overlay — le badge orange z-index:20 + position:absolute; top:0; right:0 s'affiche dans le coin supérieur droit de la carte, au-dessus du texte
  • AC-11 · /tessera test-v15 — commande enregistrée ; ouvre TesseraTestScreenV15
