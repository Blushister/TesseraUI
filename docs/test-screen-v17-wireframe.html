Acceptance Criteria — TesseraUI v1.7 (@media)

  • AC-01 · extractMediaBlocks() — blocs @media { ... } extraits du CSS avant le regex plat, remplacés par des espaces
  • AC-02 · imbrication d'accolades — profondeur comptée correctement ; un } intérieur à une règle ne termine pas le bloc @media prématurément
  • AC-03 · max-width parsed — @media (max-width: 360px) → MediaBlock.maxWidth = 360
  • AC-04 · min-width parsed — @media (min-width: 400px) → MediaBlock.minWidth = 400
  • AC-05 · MediaBlock.matches() — retourne true ssi viewport satisfait minWidth ET maxWidth ; 0 = contrainte ignorée
  • AC-06 · TesseraStyleSheet.forViewport() — retourne this si aucun bloc media ; sinon fusionne les règles des blocs matchants dans base/hover/etc.
  • AC-07 · forViewport no mediaBlocks — le stylesheet résultant n'a pas de mediaBlocks (déjà résolus) — pas de double application
  • AC-08 · renderer activation — TesseraTemplateRenderer.build() appelle template.styleSheet().forViewport(guiScaledWidth())
  • AC-09 · guiScaledWidth() fallback — retourne Integer.MAX_VALUE hors contexte Minecraft (tests unitaires)
  • AC-10 · display:none toggle — .tool-wide { display:none } dans @media (max-width: 360px) masque le widget quand viewport ≤ 360px
  • AC-11 · display:flex reveal — .wide-block { display:flex } dans @media (min-width: 400px) affiche le bloc quand viewport ≥ 400px
  • AC-12 · /tessera test-v17 — commande enregistrée, ouvre TesseraTestScreenV17
